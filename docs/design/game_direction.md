# Game Direction

Formal gameplay, content, UI, and asset decisions are tracked in `docs/design/gdd.md`. This file remains the short direction summary.

## Working Title

Cursed Curio Shop

## Positioning

A single-player first-person 3D appraisal game about running a late-night oddity shop. The player receives strange objects, inspects them with tools, chooses how to handle them, and deals with the consequences.

## Player Fantasy

The player is not a hero with a weapon. They are the last person willing to touch objects that should have stayed lost.

## Core Pillars

1. Object-first horror: the main characters are cursed objects, not monsters.
2. Clear decisions: every item should lead to a meaningful handling choice.
3. Small-shop pressure: money, reputation, and mental stability pull against each other.
4. Streamable moments: each item should produce a screenshot, short clip, or memorable reveal.
5. AI-scalable content: the game structure should allow new oddities to be added cheaply.

## MVP Scope

- One shop interior
- Ten oddities
- Three inspection tools
- Five in-game days
- One bad ending
- Basic main menu and build export

## Explicit Non-Goals For MVP

- Multiplayer
- Online economy
- Live in-game generative AI
- Large open world
- Complex human characters
- Mobile or console release
