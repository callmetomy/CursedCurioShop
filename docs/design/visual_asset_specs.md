# Visual Asset Specs

## Purpose

This document defines the first MVP visual asset pack derived from `docs/design/gdd.md`.

The current goal is not final key art. The goal is a coherent set of project-local UI and material assets that can be used directly in Godot while the gameplay loop continues to mature.

## Art Direction Constraints

- Late-night antique shop, not fantasy combat UI.
- Practical materials: paper, brass, dark wood, worn wallpaper, porcelain, wax, ink.
- Object inspection must remain readable.
- UI should support appraisal and decision-making, not dominate the screen.
- Curse markings should be sparse and tied to clues.

## Acceptance Criteria

The first pack is acceptable when:

- Assets are generated by a repeatable local command.
- PNG files exist in Godot runtime asset folders.
- UI assets use readable silhouettes.
- Texture assets tile or crop without obvious hard dependency on one scene layout.
- Transparent assets include alpha.
- The generation command is covered by tests.

## Generation Command

```powershell
python -m tools.art.generate_mvp_visual_assets --root .
```

The command is deterministic and does not call paid APIs.
